### 1. 基本姿勢を指定する

```
トップダウン思考 且つ 本質思考 且つ 抽象思考で考えて。
```

```
具体に走らず、まず抽象レベルで骨格を固めて。
```

```
「何を作るか」ではなく「なぜそれが必要か」から考えて。
```

---

### 2. 問題の再定義を求める

```
この問題の本質は何？1文で表現して。
```

```
複数の問題に見えるが、根本原因は1つに収束する？
```

```
この問題を解決すると、他に何が自動的に解決される？
```

---

### 3. 設計の骨格を求める

```
最上位の設計原則を2-3個に絞って提示して。
```

```
構成要素を最小化した抽象構造を図示して。
```

```
境界（責務の分離点）はどこに引くべき？
```

---

### 4. 選択肢の比較を求める

```
候補を3パターンまで提示。各パターンのトレードオフを明示して。
```

```
全体最適・部分最適の観点で比較して。
```

```
将来の修正コストが最小になるのはどれ？
```

---

### 5. 具体化を制御する

```
まだ具体に落とさないで。抽象レベルで合意してから進めたい。
```

```
変数名や実装詳細は後。まず日本語で設計を表現して。
```

```
この設計で「必要十分」か検証して。過剰な要素はないか？
```

---

### 6. 矛盾・冗長の検出を求める

```
矛盾点や不整合がないか全体を見直して。
```

```
この設計に無駄な要素はある？削れるものは？
```

```
この原則を適用すると、他の部分と衝突しないか？
```

---

### 7. 議論の誘導

```
私の意図を推察して、より適切な表現をブレストして。
```

```
この方向で合ってる？違和感があれば指摘して。
```

```
ステップバイステップで熟考して。急がなくていい。
```

---

### 組み合わせ例（実践的）

**設計の初期段階:**
```
トップダウン思考 且つ 本質思考 且つ 抽象思考でステップバイステップで熟考して。
まず問題の本質を1文で定義し、最上位の設計原則を2-3個に絞って提示して。
```

**選択肢の比較時:**
```
全体最適・部分最適の観点で候補を3パターンまで提示して。
将来の修正コストが最小になるのはどれ？
```

**ドキュメント化前:**
```
矛盾点や不整合がないか全体を見直して。
この設計で「必要十分」か検証して。過剰な要素はないか？
```
